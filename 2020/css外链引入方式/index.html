css外链引入方式

@import 与link的区别

  • @import方式
<style type="text/css" media="screen"> 
@import url("http://www.taobao.com/home/css/global/v2.0.css?t=20070518.css"); 
</style>
  • link方式
<link rel="stylesheet" rev="stylesheet" href="default.css" type="text/css" media="all" />

link与import的作用相同

唯一不同的就是服务对象不同

  1. @import为css服务
  2. link是为当前页面服务

@import与link的区别

  1. 根源不同

    • link属于HTML标签,而@import完全是css提供的一种方式
    • link标签除了可以加载css外,还可以做很多其它的事情，比如定义RSS，定义rel连接属性等，@import就只能加载css

  2. 加载顺序不同

    • 当一个页面被加载时侯,link引用的css会被同时加载,而@import引用的css会等到页面全部被下载完再被加载。所以有时候浏览@import加载css的页面时开始会没有样式（就是闪烁），网速慢的时候还挺明显。

  3. 使用DOM控制样式时差别

    • 当使用JavaScript控制DOM去改变样式的时候，只能使用link标签，因为@import不是DOM可以控制的。

  4. @import可以在css中再次引入其他样式表

    • 比如创建一个主样式表,在主样式表中引入其他样式表如:
    @import 'sub1.css'; 
@import 'sub2.css';
